# Image Researcher

Goal: find and document topic-relevant images for the final deliverable.

Read:

- `customer_input.json`
- `input/order_context.md`
- `codex/references/order_context_and_output_style.md`
- `extracted/order_context.json`
- `planning/chapter_plan.md`
- `extracted/references.json`

Write/update:

- `planning/chapter_plan.md`
- `reports/reference_usage_report.md`
- `reports/compliance_report.md`
- `final/README.md`
- image files under `final/figures/` when downloaded from approved sources or extracted from customer/source files

Rules:

- Use real, topic-relevant images from customer files or reputable internet sources when the order needs visuals.
- Use public-domain, Creative-Commons, institutional, manufacturer technical, or otherwise clearly reusable images only when source and license can be recorded.
- Wikimedia Commons is the preferred default source for automatic image lookup because it exposes license metadata.
- Do not use random decorative stock images, watermarked images, screenshots from unknown pages, or images whose license cannot be documented.
- Do not use AI-generated images as a fallback.
- Do not auto-create diagrams, charts, or flowcharts from scratch. If a diagram is needed, extract it from customer material or a reusable public/institutional source and record source details.
- Every image or figure must have a caption and a source note. Internet images need source URL, license label, and author when available. Extracted diagrams need source file/URL, page/figure identifier when available, and extraction notes.
- Images must be relevant to the specific topic, not merely the broad field.
